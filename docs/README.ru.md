---

Модуль `secrets-store-integration` предназначен для команд, которые хранят секреты приложений в
Deckhouse Stronghold или другом Vault-совместимом хранилище и хотят безопасно доставлять их в
нагрузки, работающие в Kubernetes.

Модуль подключает нагрузки к внешнему хранилищу секретов и помогает избежать ручного
распространения секретов между кластерами. Поддерживается доставка секретов в виде файлов и
переменных окружения.

## Основные возможности

- Доставляет секреты из Vault-совместимых хранилищ в нагрузки, работающие в Kubernetes.
- Монтирует секреты, ключи и сертификаты в контейнеры в виде файлов с помощью CSI-драйвера.
- Инжектирует секреты в процессы приложений в виде переменных окружения на этапе запуска.
- Поддерживает автоматическое подключение к локальному `stronghold` и ручное подключение к внешнему хранилищу.
- Использует аутентификацию Kubernetes `ServiceAccount` для доступа нагрузки к хранилищу секретов.
- Автоматически ротирует примонтированные секреты при изменении значения в источнике.
- Поддерживает доставку бинарных файлов в кодировке Base64 через `SecretsStoreImport`.

## Ограничения

- Модуль поддерживает только хранилища секретов, совместимые с API HashiCorp Vault.
- Ротация примонтированных секретов не работает для файлов, подключенных через `subPath`.
